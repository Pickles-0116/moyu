- **核心目标**：设计一款PC浏览器端的古风Idle/放置类摸鱼小游戏《掌柜偷闲录》，参考《叫我大掌柜》大商途玩法，浏览器打开即玩，无需部署

## 1. 产品目标（3个正交目标）

### 目标一：让用户"敢玩" — 隐蔽性零感知（体验基线）

在任意工作场景下，用户能在 **500ms 内** 通过老板键切换到伪装界面，且伪装界面与常见办公软件视觉一致，使旁观者无法辨识出游戏。

**衡量标准**：老板键响应 < 500ms；伪装界面"非游戏"判定率 ≥ 95%

### 目标二：让用户"玩得爽" — 碎片化进展感（核心体验）

每次打开游戏 **10 秒内** 获得首次有意义反馈，单次会话 2-5 分钟内至少完成 1 次升级或解锁。核心循环 15-30 秒闭合。

**衡量标准**：首次反馈 < 10s；核心循环完成率 ≥ 60%；次均时长 5-8 分钟

### 目标三：让用户"回来玩" — 离线收益与长线动力（留存引擎）

离线期间持续产生收益（在线被动收益 50-80%，上限 8 小时），通过声望重置和地图路段解锁提供长线成长动力。

**衡量标准**：D1 ≥ 35%；D7 ≥ 12%；D30 ≥ 5%；日均会话 ≥ 3 次

## 2. 用户故事（5个场景）

### US-1：碎片打工人小林（画像A）
> 作为互联网公司执行层员工，我希望在等待代码编译的2-3分钟里，打开浏览器Tab玩一会儿，听到有人走近按快捷键切回工作界面，忙完切回来发现进度还在，离线这段时间商队还在帮我赚银两。

**关键需求**：老板键 < 500ms切换、自动保存、离线收益、打开即玩

### US-2：会议生存者老王（画像B）
> 作为经常开长会的项目经理，我希望在会议中无关环节里偷偷玩，游戏看起来就像我在用Excel记会议纪要，被cue到发言时一键切回会议文档。

**关键需求**：伪装为Excel界面、不需要持续操作、随时暂停无惩罚

### US-3：午后回血型小陈（画像C）
> 作为午后容易犯困的职场人，我希望玩几分钟提提神，快速做几个选择看到资源涨，升级个店铺，玩5分钟感觉精神了就停，精力用完自然要等恢复。

**关键需求**：爽快即时反馈、精力系统限制单次时长、短时间高潮体验

### US-4：深度体验用户午休窗口
> 午休有15-30分钟自由时间，希望连续跑几轮商途、升级店铺、攒资源解锁新路段，下午工作前有"休息过了"的满足感。

**关键需求**：支撑15-30分钟深度游玩、多层次目标

### US-5：多日回归用户
> 前几天玩过，今天打开发现昨晚离线后赚了一笔银两，每日商情刷新了货物价格，还有新的江湖传闻碎片可以收集。

**关键需求**：离线收益结算、每日商情刷新、传闻碎片更新

## 3. 用户研究洞察（来自瑞思）

### 3.1 用户画像

| 画像 | 年龄 | 角色 | 核心场景 | 单次时长 | 日均频次 | 关键需求 |
|------|------|------|---------|---------|---------|---------|
| A 碎片打工人小林 | 25-30 | 普通员工 | 任务间隙 | 2-5min | 5-6次 | 进展感、快速反馈 |
| B 会议生存者老王 | 28-35 | 中层管理者 | 会议中 | 30s-5min | 2-3次 | 隐蔽性、随时暂停 |
| C 午后回血型小陈 | 22-32 | 各层级 | 午后犯困 | 3-5min | 2-3次 | 爽快感、提神效果 |

### 3.2 六条关键洞察

1. **PC浏览器摸鱼是未被满足的蓝海**：96%用手机摸鱼，仅33%用PC，需求存在但供给不足 → 产品定位PC浏览器优先
2. **摸鱼有正面心理价值**：46%因已完成工作而摸鱼，80%更放松 → 产品调性强调"高效休息工具"
3. **"被中断"是摸鱼场景本质特征**：单次均8分钟，突发中断是常态 → 自动保存+离线收益是P0
4. **碎片时间的"进展感"决定留存**：日均6-8次×2-5min → 每次会话必须有可感知的进展
5. **隐蔽性是用户"敢不敢用"的前提**：53%会延长休息继续玩 → 老板键/伪装模式是P0中的P0
6. **管理者是高频重度用户**：86%更爱玩，会议场景独特 → 伪装界面需覆盖会议场景

## 7. 核心设计详解

### 7.1 核心玩法循环

```
┌─────────────────────────────────────────────────────┐
│              核心循环（15-30秒闭合）                    │
│                                                     │
│  出发(1s) → 行进(5-10s) → 触发事件 → A/B选择(2-5s)    │
│      ↑                                    │          │
│      └──── 即时资源变化反馈(1-2s) ←───────┘          │
│                                                     │
│  被动层（持续运行）：店铺每10秒结算银两，离线也计算      │
└─────────────────────────────────────────────────────┘
```

### 7.2 四维资源系统

| 资源 | 图标 | 作用 | 产出 | 消耗 | 初始值 |
|------|------|------|------|------|--------|
| 银两 | 🪙 | 核心经济，升级店铺/买卖 | 事件/卖货/店铺产出/离线 | 升级店铺/买货/看病 | 100 |
| 声望 | 🏆 | 长线投资，Prestige重置 | 事件选择/完成商途/成就 | Prestige消耗 | 0 |
| 体力 | ❤️ | 生存约束，归零结束当轮商途 | 休息/药铺/事件 | 行进/负面事件 | 100 |
| 商途值 | 🗺️ | 进度可视化，解锁路段 | 每次行进+5/完成轮次+50 | 无（只增不减） | 0 |

### 7.3 精力系统

- 初始20点，每次行进消耗1点，5分钟恢复1点（离线也恢复）
- 20精力×15秒/次 = 5分钟一轮商途，自然限制单次时长
- 精力不足时"出发"按钮灰色+恢复倒计时，店铺被动收益不受影响

### 7.4 事件系统（15个基础事件）

事件类型覆盖：风险收益型、资源转换型、时机依赖型、赌博型、纯增益型

**示例事件**：

| 事件 | 选项A | 选项B | 设计意图 |
|------|-------|-------|---------|
| 路遇茶商 | 花80银两买龙井(可卖150-200) | 婉拒赶路(无变化) | 资源转换型 |
| 山贼拦路 | 交钱-50银两 | 拼死一搏🎲(50%声望+5/50%体力-30) | 赌博型 |
| 施粥棚前 | 捐银-30,声望+8 | 路过不停 | 名望投资型 |
| 古寺问签 | 求签🎲(70%下次收益×2/30%×0.5) | 不信此道 | 赌博+时机型 |
| 赌坊诱惑 | 赌一把🎲(30%银两×1.5/70%-100) | 不屑一顾(声望+3) | 高风险赌博 |
| 落水书生 | 下河救人(体力-20,声望+12) | 扔绳(银两-10,声望+5) | 三选项增加深度 |

### 7.5 老板键/伪装模式（核心差异化）

- 快捷键：`Esc`（默认，可自定义）
- 默认伪装为Excel财务表格（行号列标+工具栏+合理数据）
- 切换<500ms，伪装时游戏暂停但店铺产出和精力恢复不暂停
- P1扩展：记事本/代码编辑器/邮件客户端

### 7.6 店铺系统

| 店铺 | 解锁条件 | 初始产出 | 特殊效果 |
|------|---------|---------|---------|
| 茶馆 | 默认 | 10银两/10s | 无 |
| 布庄 | 银两≥500 | 25银两/10s | 无 |
| 粮铺 | 银两≥2000 | 60银两/10s | 无 |
| 酒楼 | P1解锁 | 150银两/10s | 每5级+1精力上限 |
| 药铺 | P1解锁 | 80银两/10s | 每5级体力上限+10 |
| 镖局 | P1解锁 | 200银两/10s | 每5级离线收益+5% |

里程碑：10级×2、25级×3、50级×5、100级×10

### 7.7 声望重置（Prestige）

- 门槛：500声望（约10-15小时游玩）
- 1声望=1传承点=全店铺产出+1%（永久）
- 每次重置提升离线上限+1h、精力上限+2
