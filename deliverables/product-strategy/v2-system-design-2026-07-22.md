## 1. 囤货交易系统正式化

### 1.1 系统定位

囤货交易是用户明确要求保留的核心玩法。当前v1.1已实现基本功能（4小时刷新6种货物、持仓99/种、无手续费、波动×0.5~×2.5），但经济平衡存在严重问题——中期套利占产出37%。

**v2目标**：落实数析建议的四重约束，将日均套利从~1,070,000降至~119,000（降幅89%），同时保留"低买高卖"的核心乐趣。

### 1.2 四重约束参数（修改现有）

| 约束 | 当前值 | v2值 | 变更说明 | 游戏配置键 |
|------|--------|------|---------|-----------|
| 持仓上限 | 99个/种 | **30个/种** | 限制单次投入资本，-70% | `marketMaxHolding: 30` |
| 刷新频率 | 4小时(6次/天) | **6小时(4次/天)** | 日操作次数-33% | `marketRefreshInterval: 21600` |
| 交易手续费 | 0% | **15%（买卖各扣7.5%）** | 净利润降低~40% | `marketTradeFee: 0.15` |
| 价格波动区间 | ×0.5~×2.5 | **×0.6~×2.0** | 价差从2.0收窄至1.4 | `marketVolatilityMin: 0.6` / `marketVolatilityMax: 2.0` |

### 1.3 手续费计算逻辑

```javascript
// 买入手续费
function calculateBuyFee(totalCost) {
  const fee = Math.floor(totalCost * 0.075); // 7.5%
  return fee;
}
// 买入总花费 = 货物价格 × 数量 + 手续费
// totalCost = price * qty + fee

// 卖出手续费
function calculateSellFee(totalIncome) {
  const fee = Math.floor(totalIncome * 0.075); // 7.5%
  return fee;
}
// 卖出实际收入 = 货物价格 × 数量 - 手续费
// actualIncome = price * qty - fee
```

**UI展示**：交易面板中显示"价格 × 数量 = 小计 / 手续费(7.5%) / 实付银两"，让玩家明确感知手续费的存在。

### 1.4 价格波动调整

```javascript
// 修改 refreshMarket() 方法
refreshMarket() {
  const prices = {};
  for (const good of DAILY_MARKET_GOODS) {
    // 原: const mult = 0.5 + Math.random() * 2.0;  // ×0.5~×2.5
    const mult = 0.6 + Math.random() * 1.4;  // ×0.6~×2.0
    prices[good.id] = {
      price: Math.floor(good.basePrice * mult),
      multiplier: parseFloat(mult.toFixed(2)),
      name: good.name, icon: good.icon, basePrice: good.basePrice
    };
  }
  // ... 其余逻辑不变
}
```

### 1.5 保留的核心乐趣

- ✅ 低买高卖的核心决策仍在（×0.6~×2.0仍有足够价差）
- ✅ 6种货物种类不变，保留多样化交易策略
- ✅ 刷新机制不变（定时刷新+可查看剩余时间）
- ✅ 30个持仓仍有操作空间，限制的是单次投入而非玩法本身
- ✅ 手续费让"频繁交易"不再是零成本，但"等待好行情再操作"仍有利可图

### 1.6 与黑市的错峰设计

| 系统 | 刷新间隔 | 错峰策略 |
|------|---------|---------|
| 商情交易 | 6小时 | 00:00/06:00/12:00/18:00（整点刷新） |
| 黑市 | 6小时 | 03:00/09:00/15:00/21:00（半点刷新，与商情错开3小时） |

**设计意图**：不同时段有不同的"需要回来看一眼"的理由，增加日活频次。

- 地点操作不消耗体力（体力仅商途行进消耗）
- 伪装模式下不可前往地点

### 2.1 酒楼——声望提升中心

#### 2.1.1 宴请系统

**核心功能**：花银两举办宴会，获得声望。这是银两→声望的直接转化通道。

| 宴会档次 | 银两花费 | 获得声望 | 性价比(声望/千银两) | 适合阶段 |
|---------|---------|---------|---------------------|---------|
| 小宴 | 500 | 10-15 | 20-30 | 前期 |
| 中宴 | 2,000 | 30-40 | 15-20 | 中期 |
| 大宴 | 8,000 | 80-100 | 10-12.5 | 后期 |

**声望获得区间说明**：声望在范围内随机（如小宴10-15声望，每次随机取值），保留一定不确定性，增加"开出大奖"的惊喜感。

#### 2.1.2 递增定价机制

落实数析建议，声望购买采用递增定价：

```
单次价格 = 基础单价 × (1 + 当日已购次数 × 0.10) × Prestige系数 × 声望进度系数
```

| 系数 | 公式 | 说明 |
|------|------|------|
| 基础单价 | 声望/银两换算 | 小宴=500银两/12.5声望=40银两/声望 |
| 递增系数 | ×(1 + 当日已宴次数 × 0.10) | 第1次×1.0，第2次×1.1，第3次×1.2... |
| Prestige系数 | ×(1 + Prestige次数 × 0.5) | 第1轮×1.0，第2轮×1.5，第3轮×2.0 |
| 声望进度系数 | ×(1 + 当前声望 / 200) | 声望0时×1.0，声望200时×2.0 |

**每日限次**：每日限宴 **1次**（防止声望速通），消耗 **2精力**。

#### 2.1.3 各阶段宴请成本示例

| 阶段 | Prestige轮次 | 当前声望 | 小宴实际花费 | 中宴实际花费 | 大宴实际花费 |
|------|-------------|---------|------------|------------|------------|
| 前期 | 0 | 0 | 500 | 2,000 | 8,000 |
| 前期 | 0 | 100 | 750 | 3,000 | 12,000 |
| 中期 | 1 | 200 | 1,500 | 6,000 | 24,000 |
| 后期 | 2 | 400 | 4,500 | 18,000 | 72,000 |

**设计意图**：随着游戏进度推进，宴请成本递增，确保后期仍为有效的银两消耗出口。

#### 2.1.4 交互流程

```
点击🏯外出 → 选择🍶酒楼 → 进入酒楼面板
    ↓
显示三档宴会(小宴/中宴/大宴)
显示当前价格(含递增/Prestige/声望系数)
显示今日剩余次数(0/1)
    ↓
选择档次 → 确认花费 → 银两扣除 → 精力-2
    ↓
宴会动画(1-2秒：觥筹交错) → 声望飘字(+12~15)
    ↓
日志："你在酒楼设下小宴，宾客尽欢，声望+12！"
    ↓
今日次数变为(1/1)，按钮灰色禁用
```

#### 2.1.5 古风文案设计

- 酒楼名称：**醉仙楼**
- 小宴文案：「设下清茶小宴，邀三五好友品茗论道」

### 2.2 御街——随机事件地点

#### 2.2.1 系统定位

御街是混合赌运气地点，消耗1精力，可能获得声望/银两/体力，也有负面结果。定位为**中风险中回报的声望补充渠道**。

#### 2.2.2 御街随机事件表

| 事件ID | 名称 | 概率 | 效果 | 类型 | 文案 |
|--------|------|------|------|------|------|
| ys_01 | 偶遇贵人 | 10% | 声望+30 | 正面 | 「御街上一位朝中大臣驻足与你攀谈，对你赞不绝口。」 |
| ys_02 | 路遇施舍 | 15% | 声望+15 | 正面 | 「你见路边老妪乞讨，施舍了几两碎银，旁人纷纷称赞。」 |
| ys_03 | 拾金不昧 | 8% | 声望+20, 银两-50 | 正面 | 「你捡到一袋银两，原地等候失主归还，失主感激不已。」 |
| ys_04 | 意外之财 | 10% | 银两+200 | 正面 | 「路边的商贩认出你是大掌柜，硬塞给你一包样品。」 |
| ys_05 | 体力恢复 | 12% | 体力+30 | 正面 | 「御街花圃清香沁人，你精神一振，体力恢复了不少。」 |
| ys_06 | 街头杂耍 | 10% | 体力+15, 银两-30 | 正面 | 「你驻足观看杂耍，赏了几个铜板，心情大好。」 |
| ys_07 | 无事发生 | 15% | 无变化 | 中性 | 「你在御街闲逛了一圈，没什么特别的发现。」 |
| ys_08 | 遭遇扒手 | 8% | 银两-100 | 负面 | 「人群拥挤，你回头看时钱袋已不见了！」 |
| ys_09 | 被人诬陷 | 7% | 声望-10 | 负面 | 「有人指认你撞了他，你百口莫辩，旁人议论纷纷。」 |
| ys_10 | 官差盘查 | 5% | 银两-150, 体力-10 | 负面 | 「官差拦住你盘查良久，耽误了不少工夫。」 |

**概率验证**：正面55% + 中性15% + 负面30% = 100%

**期望值分析**：
- 声望期望 = 0.10×30 + 0.15×15 + 0.08×20 + 0.07×(-10) = 3.0 + 2.25 + 1.6 - 0.7 = **+6.15声望/次**
- 银两期望 = 0.08×(-50) + 0.10×200 + 0.06×(-30) + 0.08×(-100) + 0.05×(-150) = -4 + 20 - 1.8 - 8 - 7.5 = **-1.3银两/次**
- 体力期望 = 0.12×30 + 0.10×15 + 0.05×(-10) = 3.6 + 1.5 - 0.5 = **+4.6体力/次**

**设计评估**：御街是声望的正期望获取渠道（+6.15/次），银两基本持平（-1.3/次），体力微正。考虑到消耗1精力，与酒楼宴请（500银两→12.5声望=40银两/声望）相比，御街更经济但有声望损失风险。

```
    ↓
```

### 2.3 赌坊——独立赌博系统

#### 2.3.0 系统概述

赌坊是独立赌博玩法入口，包含三种玩法：掷骰子、押注竞猜、赌石。声望≥100解锁。

**统一限制参数**（落实数析建议）：

| 参数 | 值 | 说明 |
|------|------|------|
| 期望值(EV) | **0.90** | 10%庄家优势，负期望消耗 |
| 单次最低赌注 | 1,000银两 | 前期可参与 |
| 单次最高赌注 | 100,000银两 | 限制单次冲击 |
| 日累计上限 | 2,000,000银两 | 防富豪无意义消耗 |
| 日限次 | **20次/天**（三玩法共享） | 防沉迷+控制节奏 |
| 解锁条件 | 声望≥100 | 中期才开放 |
| 精力消耗 | 1精力/次 | 与地点系统联动 |

#### 2.3.1 玩法A：掷骰子猜大小

**规则**：
- 3颗骰子（1-6），总和3-18
- 猜大（11-18）或猜小（3-10），赔率 **1:1**
- 豹子（三个相同，如111/222...666），赔率 **1:8**（需额外押注豹子）
- 豹子时大小猜注一律判负

**下注/结算逻辑**：

```javascript
function playDice(betType, betAmount) {
  // betType: 'big' | 'small' | 'triple'
  // betAmount: 1000 ~ 100000
  const dice = [
    1 + Math.floor(Math.random() * 6),
    1 + Math.floor(Math.random() * 6),
    1 + Math.floor(Math.random() * 6)
  ];
  const sum = dice[0] + dice[1] + dice[2];
  const isTriple = dice[0] === dice[1] && dice[1] === dice[2];

  if (isTriple) {
    // 豹子：大小猜注判负，豹子猜注赢8倍
    if (betType === 'triple') {
      return { win: true, payout: betAmount * 9, dice, sum, result: 'triple' };
      // 返回9倍（含本金），净赚8倍
    } else {
      return { win: false, payout: 0, dice, sum, result: 'triple_lose' };
      // 大小猜注全损
    }
  }

  // 非豹子：豹子猜注判负
  if (betType === 'triple') {
    return { win: false, payout: 0, dice, sum, result: 'no_triple' };
  }

  // 大小判断
  const isBig = sum >= 11;
  const guessedRight = (betType === 'big' && isBig) || (betType === 'small' && !isBig);
  if (guessedRight) {
    return { win: true, payout: betAmount * 2, dice, sum, result: isBig ? 'big' : 'small' };
    // 返回2倍（含本金），净赚1倍
  } else {
    return { win: false, payout: 0, dice, sum, result: isBig ? 'big' : 'small' };
  }
}
```

**概率与EV验证**：
- 总组合数：6³ = 216
- 豹子组合：6种 → P(豹子) = 6/216 = 2.78%
- 大/小各105种（排除豹子） → P(大) = P(小) = 105/216 = 48.61%
- EV(猜大小) = 0.4861 × 2 = **0.972**（略高于0.90目标，但豹子判负抵消部分）
- 综合EV(猜大小，含豹子判负) = (1 - 0.0278) × 0.4861 × 2 = 0.972 × 0.9722 = **0.945**
- EV(猜豹子) = 0.0278 × 9 = **0.250**（极低EV，高风险高回报）

**综合EV**：假设玩家70%猜大小、30%猜豹子：
- EV = 0.7 × 0.945 + 0.3 × 0.250 = 0.6615 + 0.075 = **0.737**

**调整说明**：综合EV偏低(0.737)，因为豹子赌注EV极低。但赌石和竞猜的EV会拉高整体。建议掷骰子豹子赔率调整为1:10（净赚9倍），则EV(豹子)=0.278×10=0.278，仍为负期望但稍合理。最终统一EV由三种玩法的使用频率决定，目标0.90。

#### 2.3.2 玩法B：押注竞猜

**规则**：
- 每日刷新1个竞猜题目（与商途系统联动）
- 押注银两，猜对 **2倍返还**（净赚1倍），猜错全损
- 每日限1次

**竞猜题目池**（每日随机选1个）：

| 题目ID | 题目 | 正面条件 | 概率 |
|--------|------|---------|------|

```javascript
canExpandBackpack() {
  const currentSlots = this.state.backpack.maxSlots;
  if (currentSlots >= GAME_CONFIG.backpackMaxSlots) return false;
  const expandIdx = (currentSlots - GAME_CONFIG.backpackInitialSlots) / GAME_CONFIG.backpackExpandStep;
  const cost = GAME_CONFIG.backpackExpandCosts[expandIdx] || 2000;
  const repReq = GAME_CONFIG.backpackExpandRepReq[expandIdx] || 400;
  return this.state.resources.silver >= cost && this.state.resources.reputation >= repReq;
}

expandBackpack() {
  if (this.state.backpack.maxSlots >= GAME_CONFIG.backpackMaxSlots) return false;
  const expandIdx = (this.state.backpack.maxSlots - GAME_CONFIG.backpackInitialSlots) / GAME_CONFIG.backpackExpandStep;
  const cost = GAME_CONFIG.backpackExpandCosts[expandIdx] || 2000;
  const repReq = GAME_CONFIG.backpackExpandRepReq[expandIdx] || 400;
  if (this.state.resources.silver < cost) return false;
  if (this.state.resources.reputation < repReq) return false;
  this.modifyResource('silver', -cost);
  this.state.backpack.maxSlots += GAME_CONFIG.backpackExpandStep;
  this.addLog(`📦 背包扩容至 ${this.state.backpack.maxSlots} 格！`, 'success');
  this.emit('backpackChange');
  this.autoSave();
  return true;
}
```

## 4. 产业投资系统

### 4.0 系统定位

基于竞析建议5，设计简化版产业投资，作为后期银两消耗出口。分为两部分：店铺增量产出（稳健）+ 盐场投资（高风险高回报）。

### 4.1 店铺每级增量产出（P0级修复）

#### 问题

当前店铺产出仅在里程碑等级（Lv.10/25/50/100）跳变，里程碑之间升级零产出收益。数析诊断这是"银两通胀的根源"——玩家停在里程碑不升级，产出90%+变盈余。

#### 解决方案

落实数析建议，每级增加固定增量产出（加算而非乘算）：

```javascript
// 修改 getShopOutput() 方法
getShopOutput(shopId) {
  const shop = this.state.shops[shopId];
  if (!shop || !shop.unlocked) return 0;
  const shopDef = SHOPS[shopId];

  // 里程碑倍率产出（原有逻辑）
  let milestoneOutput = shopDef.baseOutput * shop.level;
  for (const [level, mult] of Object.entries(GAME_CONFIG.milestoneMultipliers)) {
    if (shop.level >= parseInt(level)) {
      milestoneOutput = Math.max(milestoneOutput, shopDef.baseOutput * parseInt(level) * mult);
    }
  }

  // v2新增：每级增量产出（加算）
  const incrementalOutput = shopDef.baseOutput * 0.02 * (shop.level - 1);

  let output = milestoneOutput + incrementalOutput;
  output = Math.floor(output * this.state.journey.shopOutputBoost);

  // Prestige legacy加成（原有逻辑）
  if (this.state.prestige.legacy > 0) {
    output = Math.floor(output * (1 + this.state.prestige.legacy * GAME_CONFIG.legacyPerReputation));
  }
  return Math.floor(output);
}
```

#### 增量产出效果对比

| 等级 | 原产出(base×倍率) | v2产出(倍率+增量) | 增量占比 | 提升 |
|------|-----------------|-----------------|---------|------|
| Lv.10 | base × 2 = 2.0base | 2.0base + 0.18base = 2.18base | 8.3% | +9% |
| Lv.25 | base × 3 = 3.0base | 3.0base + 0.48base = 3.48base | 13.8% | +16% |
| Lv.50 | base × 5 = 5.0base | 5.0base + 0.98base = 5.98base | 16.4% | +20% |
| Lv.100 | base × 10 = 10.0base | 10.0base + 1.98base = 11.98base | 16.5% | +20% |

**升级回本周期验证**（以茶馆base=10为例）：
- Lv.10→11：增量产出=10×0.02×1=0.2银两/10s=1,728/天；升级费用≈800银两；回本≈0.5天 ✅
- Lv.50→51：增量产出=10×0.02×1=0.2银两/10s=1,728/天；升级费用≈30,000银两；回本≈17天 ✅
- Lv.99→100：增量产出=10×0.02×1=0.2银两/10s=1,728/天；升级费用≈200,000银两；回本≈115天 ⚠️

**调整说明**：Lv.99→100回本周期过长(115天)，但Lv.100有里程碑倍率从×5跳到×10的巨大产出提升，这才是升级动力。增量产出是解决"非里程碑等级无收益"的问题，不改变里程碑跳变的核心激励。

### 4.2 盐场投资系统（新增，高风险高回报）

#### 4.2.1 系统概述

借鉴竞析的大商途盐场机制，设计简化版盐场投资。盐场是后期大额银两消耗出口+高回报引擎。

#### 4.2.2 解锁条件

| 条件 | 要求 | 说明 |
|------|------|------|
| 声望 | ≥300 | 后期解锁 |
| Prestige | ≥1 | 至少完成1次声望重置 |
| 银两 | ≥50,000 | 起步资金门槛 |

#### 4.2.3 核心机制

**购买盐场**：
- 盐场基础价格：50,000银两/个
- 每购买1个，下一个价格+20%（第1个50K，第2个60K，第3个72K...）

**声望限制名额**：

| 声望 | 可拥有盐场上限 |
|------|-------------|
| 300 | 1个 |
| 500 | 2个 |
| 800 | 3个 |
| 1200 | 4个 |

**开采速度选择**：

| 速度 | 产出倍率 | 查封风险/分钟 | 适合策略 |
|------|---------|-------------|---------|
| 慢速 | ×1.0 | +1%/分钟 | 稳健长期 |
| 中速 | ×1.5 | +2%/分钟 | 平衡 |
| 快速 | ×2.0 | +3%/分钟 | 激进短期 |
| 极速 | ×3.0 | +5%/分钟 | 赌命冲刺 |

**产出公式**：
```javascript
// 盐场每10秒结算（与店铺同步）
function getSaltFieldOutput(saltField) {
  const baseOutput = 500; // 基础产出500银两/10s
  const speedMult = { slow: 1.0, medium: 1.5, fast: 2.0, extreme: 3.0 };
  return Math.floor(baseOutput * speedMult[saltField.speed] * saltField.level);
}
```

**查封机制**：
- 查封概率 = 累积风险值（每分钟增加对应速度的风险值）
- 当累积风险值达到阈值（100%）时，盐场被查封
- 被查封：盐场消失，损失全部投入（包括购买费用和已产出但未领取的银两）
- **主动收手**：玩家可随时选择"收手退出"，回收50%购买费用

**查封风险衰减**：
- 降低开采速度时，累积风险值按比例衰减（慢速时每分钟-2%）
- 完全停止开采时，风险值每分钟-5%

#### 4.2.4 盐场状态管理

```javascript
  purchasePrice: 50000,       // 购买价格
  speed: "medium",            // slow/medium/fast/extreme

  totalOutput: 0,             // 累计产出

  active: true,               // 是否在开采中
}
```

| 慢速 | 432,000 | ~100分钟(1.7h) | ~720,000 | +220,000 |

### 5.5 终局银两供奉系统（P2）

#### 问题描述

满级后日产出~16M，现有消耗渠道(赌坊200K+黑市800K+盐场200K)日均仅~1.2M，余额率89%。需要终局银两出口。

#### 设计方案

**银两供奉系统**：满级（全部店铺Lv.100）后解锁，向NPC"商神"供奉大量银两换取Prestige点数。

| 供奉档次 | 银两花费 | 获得传承点 | 等效声望 | 说明 |
|---------|---------|-----------|---------|------|
| 小奉 | 1,000,000 | 10 | 10声望 | 等效Prestige重置10声望 |
| 中奉 | 5,000,000 | 60 | 60声望 | 性价比提升20% |
| 大奉 | 20,000,000 | 300 | 300声望 | 性价比提升50% |
| 巨奉 | 100,000,000 | 2,000 | 2,000声望 | 终极供奉 |

**设计说明**：
- 供奉获得的是传承点（等效声望重置的传承点），但不触发Prestige重置
- 供奉不清零银两/店铺/商途值，是"额外获取传承点"的渠道
- 满级玩家通过供奉积累传承点，提升产出加成，形成"满级→供奉→更强产出→更多供奉"的循环
- 巨奉(100M)需要约6天满级产出来积攒，是真正的长线目标

## 7. Non-goals更新

在现有Non-goals基础上，新增以下v2 Non-goals：

### 7.1 赌坊相关

- ❌ **不做真实多人赌博/对战赌博**（单机无社交，所有赌博均为玩家vs庄家）
- ❌ **不做赌坊排行榜/赌博成就**（避免鼓励过度赌博）
- ❌ **不做赌坊VIP/会员系统**（不引入付费赌博机制）
- ❌ **不做借贷/赊账赌博**（玩家只能用自有银两下注）

### 7.2 黑市相关

- ❌ **不做黑市玩家间交易**（单机无社交，黑市仅NPC交易）
- ❌ **不做黑市拍卖/竞价**（增加操作复杂度，不适合碎片化摸鱼）

### 7.3 盐场相关

- ❌ **不做盐场完整复杂版**（大商途的挚友看护/多盐场组合/盐场等级等不做，仅保留核心的购买→开采→查封/收手机制）
- ❌ **不做盐场与门客联动**（无门客系统）

### 7.4 声望相关

- ❌ **不做声望卖钱**（大商途后期可卖声望换钱，但我们的声望更稀缺，不宜双向流通。声望是"投资"而非"可变现资产"）
- ❌ **不做声望交易/赠送**（单机无社交）

### 7.5 终局相关

- ❌ **不做无限等级**（店铺上限Lv.100，满级后通过供奉系统+Prestige循环维持长线动力）

### 10.1 囤货交易

```

Then 每种货物最多持有30个

Given 玩家买入10个丝绸(单价100)
When 计算实付金额
Then 总价=100×10=1000，手续费=1000×7.5%=75，实付=1075

Given 玩家卖出10个丝绸(单价200)
When 计算实收金额
Then 总价=200×10=2000，手续费=2000×7.5%=150，实收=1850

Given 商情已刷新
When 查看下次刷新时间
Then 间隔为6小时(非4小时)

Given 商情价格波动
When 查看倍率
Then 范围在×0.6~×2.0之间(非×0.5~×2.5)
```

## 11. 开放问题

1. **赌坊EV微调**：掷骰子综合EV=0.737（因豹子EV极低），是否将豹子赔率从1:8调整为1:10？——建议v2.1上线后观察玩家豹子下注频率和数据决定
2. **盐场查封风险公式**：当前设计为线性累积(每分钟+固定值)，是否需要改为非线性(越快越后期风险增速越快)？——建议先用线性版本上线，根据玩家反馈调整
3. **黑市令消耗规则**：黑市令是"消耗品"(每次进入消耗1个)还是"永久钥匙"(拥有即可进入)？——建议为消耗品，增加稀缺感和获取动机，每次进入消耗1个
4. **供奉系统是否需要Prestige≥3**：当前设计仅需全店Lv.100，是否需要额外Prestige门槛？——建议不需要，全店Lv.100已经是足够高的门槛
5. **御街事件是否需要保底**：连续3次负面结果后是否提升正面概率？——建议不需要，30%负面概率本身不高，且御街是可选行为
6. **盐场产出是否受离线收益影响**：离线时盐场是否继续产出？——建议离线时盐场暂停(不产出也不累积风险)，避免离线回来发现盐场被查封的负面体验
