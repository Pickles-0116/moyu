## 2. 各版本详细规划

### 2.1 v0.5 Alpha — 核心循环验证

| 项目 | 内容 |
|------|------|
| **主题** | 核心循环验证 |
| **目标** | 验证"行进→事件→A/B选择→资源变化"循环是否好玩 |
| **工期** | 单人~3周 / 双人~2周 |
| **人日** | 13人日 |
| **需求** | P0-03(核心循环) + P0-04(四维资源) + P0-05(精力系统) + P0-08(事件系统15个) |

**交付物**：
- 可运行的核心循环Demo
- 15个古风事件
- 四维资源面板
- 精力系统

**验收标准**：
- [ ] 内部体验核心循环≥5分钟不无聊
- [ ] A/B选择有明确的意义感（两个选项各有优劣）
- [ ] 核心循环15-30秒闭合
- [ ] 精力系统有效限制单次时长

**主要风险**：核心循环不好玩 → 需要调整事件设计或反馈强度

**交付物**：
- 囤货交易四重约束（持仓30、刷新6h、手续费15%、波动×0.6~×2.0）
- 店铺每级增量产出（+2% base/级）
- 地点系统统一入口（🏯外出按钮 + 5地点面板）
- 酒楼宴请系统（三档 + 递增定价 + 日限1次）
- 背包扩容声望门槛（50/200/400）
- 离线收益调整（50%/6h，Prestige后恢复60%/8h）

**开发顺序**：
1. V2-02 店铺增量产出（1d）— P0级修复，影响所有后续平衡
2. V2-01 囤货四重约束（2d）— 与增量产出配合
3. V2-12 离线收益调整（1d）— 配合产出端调整
4. V2-11 背包扩容声望门槛（1d）— 独立修改
5. V2-03 地点系统框架（2d）— v2.1/v2.2的地基
6. V2-04 酒楼宴请（3d）— 地点系统第一个功能

**验收标准**：
- [ ] 囤货持仓上限30个/种，刷新6h，手续费15%正确计算
- [ ] 价格波动×0.6~×2.0
- [ ] 店铺产出含增量产出（Lv.11产出=base×2+base×0.02×10）
- [ ] 地点系统5地点面板可显示，精力联动正确
- [ ] 酒楼宴请三档+递增定价+日限1次
- [ ] 背包扩容双条件判断（银两+声望）
- [ ] 离线收益50%/6h（Prestige后60%/8h）
- [ ] **银两周转率0.80-0.90**
- [ ] **囤货套利占产出比<10%（中期）**

**主要风险**：经济参数需数据验证 → 上线后密切监控，准备hotfix

## 6. 验证检查点

### 6.1 v2.0经济修复验证（第22周末）

**必须达标的指标**：
- [ ] 囤货套利占中期产出比 < 10%（v1: 37%）
- [ ] 银两周转率（前期）≥ 0.80
- [ ] 银两周转率（中期）≥ 0.70
- [ ] 店铺Lv.11产出 > Lv.10产出（增量产出生效）
- [ ] 离线收益比例50%（Prestige后60%）

**如果未达标**：
- 套利仍>15% → 进一步降低持仓上限(30→20)或提高手续费(15%→20%)
- 周转率<0.70 → 检查消耗出口是否足够，考虑提前上线v2.1部分功能

### 6.2 v2.1玩法扩展验证（第27周末）

**必须达标的指标**：
- [ ] 赌坊综合EV ≈ 0.90（允许±0.05偏差）
- [ ] 赌坊日参与率 ≥ 30%（有赌坊权限的玩家）
- [ ] 黑市购买频率 ≥ 1次/刷新周期（有黑市权限的玩家）
- [ ] 御街使用率 ≥ 20%（有声望≥50的玩家）

**如果未达标**：
- 赌坊EV偏低 → 调整豹子赔率或赌石品质概率
- 参与率低 → 降低赌注下限(1000→500)或增加每日免费次数
- 黑市购买率低 → 调整商品定价或增加T1商品刷新概率

### 6.3 v2.2终局深化验证（第32周末）

**必须达标的指标**：
- [ ] 满级玩家银两周转率 ≥ 0.10（v1: ~0.01）
- [ ] 盐场使用率 ≥ 40%（有盐场权限的玩家）
- [ ] 供奉系统使用率 ≥ 20%（满级玩家）
- [ ] D30留存 ≥ 5%

**如果未达标**：
- 周转率<0.05 → 增加供奉性价比或新增终局消耗渠道
- 盐场使用率低 → 降低起步价格(50K→30K)或提升慢速收益
- 供奉使用率低 → 降低小奉门槛(1M→500K)或提升传承点性价比

## 8. v2新增系统与现有系统的交互关系

```
                    ┌──────────────┐
                    │   核心循环    │
                    │ (行进→事件)   │
                    └──────┬───────┘
                           │
              ┌────────────┼────────────┐
              ↓            ↓            ↓
        ┌─────────┐  ┌──────────┐  ┌──────────┐
        │ 店铺系统  │  │ 商情交易  │  │ 地点系统  │
        │ (被动产出)│  │ (囤货交易)│  │ (主动活动)│
        └────┬────┘  └────┬─────┘  └────┬─────┘
             │            │             │
             │            │    ┌────────┼────────┐
             │            │    ↓        ↓        ↓
             │            │ ┌──────┐ ┌──────┐ ┌──────┐
             │            │ │ 酒楼  │ │ 赌坊  │ │ 黑市  │
             │            │ │(声望) │ │(赌博) │ │(稀有) │
             │            │ └──┬───┘ └──┬───┘ └──┬───┘
             │            │    │        │        │
             ↓            ↓    ↓        ↓        ↓
        ┌─────────────────────────────────────────────┐
        │              银两 ←→ 声望 转化网络            │
        │  酒楼: 银两→声望  御街: 精力→声望/银两/体力  │
        │  赌坊: 银两→银两(负EV)  黑市: 银两→道具      │
        │  医馆: 银两→体力  盐场: 银两→银两(高风险)    │
        │  供奉: 银两→传承点  扩容: 银两+声望→背包格   │
        └─────────────────────────────────────────────┘
                           │
              ┌────────────┼────────────┐
              ↓            ↓            ↓
        ┌─────────┐  ┌──────────┐  ┌──────────┐
        │ Prestige │  │  供奉系统  │  │  终局循环  │
        │ (声望→   │  │ (银两→   │  │ (满级→   │
        │  传承点)  │  │  传承点)  │  │  供奉→   │
        └─────────┘  └──────────┘  │  循环)   │
                                    └──────────┘
```

**关键交互**：
1. **精力池共享**：商途行进与地点系统共享精力，形成"跑商途 vs 去地点"的选择压力
2. **银两-声望转化**：酒楼(银两→声望) + 御街(精力→声望) + 黑市查抄(损失银两+声望补偿) = 多维声望获取
3. **黑市-赌坊联动**：黑市原石在赌坊切开，黑市令控制黑市准入
4. **盐场-离线联动**：盐场离线暂停，不触发离线收益计算
5. **供奉-Prestige联动**：供奉获得传承点但不触发重置，与Prestige形成互补
